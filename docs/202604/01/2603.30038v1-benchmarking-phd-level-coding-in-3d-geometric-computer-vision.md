## 摘要
人工智能辅助编码已迅速重塑了软件实践与研究流程，然而当今的模型在生成复杂三维几何视觉的正确代码方面仍面临困难。如果模型能够可靠地编写此类代码，我们研究领域的面貌将发生重大改变。为了衡量实现这一目标的进展，我们推出了GeoCodeBench——一个评估三维视觉编码能力的博士级基准测试。每个问题都是一个填空式函数实现任务，均选自近期顶级会议的代表性论文：我们首先让工具从官方代码库中提取候选函数，然后通过细致的人工筛选来选定核心的三维几何组件。针对每个目标函数，我们生成了多样化的边界情况单元测试，从而实现完全自动化、可复现的评分。我们评估了八个具有代表性的开源与闭源模型，以反映当前生态系统的状况。表现最佳的模型GPT-5仅获得36.6%的通过率，揭示了当前能力与可靠的三维科学编码之间存在巨大差距。GeoCodeBench将任务组织成一个两级层次结构：通用三维能力（几何变换与力学/光学公式化）与研究能力（新颖算法实现与几何逻辑路由）。这些维度上的得分呈正相关，但面向研究的任务明显更难。上下文消融实验进一步表明，“提供更多论文文本”并不总是更好：在方法部分截断输入在统计上优于提供全文输入，这突显了长上下文科学理解方面尚未解决的挑战。综上所述，这些发现使GeoCodeBench成为一个严谨的测试平台，旨在推动从通用编码迈向可信赖的三维几何视觉编码的发展。

## Abstract
AI-assisted coding has rapidly reshaped software practice and research workflows, yet today's models still struggle to produce correct code for complex 3D geometric vision. If models could reliably write such code, the research of our community would change substantially. To measure progress toward that goal, we introduce GeoCodeBench, a PhD-level benchmark that evaluates coding for 3D vision. Each problem is a fill-in-the-function implementation task curated from representative papers at recent venues: we first let a tool propose candidate functions from official repositories, then perform careful human screening to select core 3D geometric components. For every target, we generate diverse, edge-case unit tests, enabling fully automatic, reproducible scoring. We evaluate eight representative open- and closed-source models to reflect the current ecosystem. The best model, GPT-5, attains only 36.6% pass rate, revealing a large gap between current capabilities and dependable 3D scientific coding. GeoCodeBench organizes tasks into a two-level hierarchy: General 3D capability (geometric transformations and mechanics/optics formulation) and Research capability (novel algorithm implementation and geometric logic routing). Scores are positively correlated across these axes, but research-oriented tasks are markedly harder. Context ablations further show that "more paper text" is not always better: cutting off at the Method section statistically outperforms full-paper inputs, highlighting unresolved challenges in long-context scientific comprehension. Together, these findings position GeoCodeBench as a rigorous testbed for advancing from generic coding to trustworthy 3D geometric vision coding.

## 1. 核心问题与整体含义
*   **研究动机**：尽管大语言模型（LLMs）在通用代码生成方面已相当成熟，但在需要精确数学、物理知识和复杂空间推理的**三维几何计算机视觉（3D Geometric CV）领域**，其生成正确、可靠代码的能力仍严重不足。这限制了AI在加速3D视觉研究（如自动原型构建、算法民主化）方面的潜力。
*   **核心目标**：为了系统性地衡量LLMs在该领域的真实编码能力，并推动其发展，本文提出了 **GeoCodeBench**——首个面向“博士级”难度的、基于执行的3D几何视觉代码生成基准测试。它旨在回答一个根本性问题：当给予与人类研究者相同的文本上下文时，LLMs实现真实3D视觉代码的能力如何？

## 2. 方法论
*   **核心思想**：模拟真实研究场景，构建一个“读论文-填代码-过测试”的评估框架。任务直接来源于近期顶级会议（CVPR/ICCV/ICLR 2025）的论文及其官方代码库。
*   **关键技术细节与流程**：
    1.  **任务构建**：
        *   **数据源**：从47篇近期顶级论文及其开源仓库中筛选。
        *   **函数筛选**：首先使用AI工具（Cursor）自动从仓库中提取候选函数（10-20个/仓库），然后由领域专家人工审核，根据“是否为核心算法”、“是否与几何相关”、“长度是否合适”等标准，最终为每个仓库保留3-5个高质量函数作为评估目标。
        *   **任务形式**：提供结构化论文内容（使用MinerU OCR工具提取并组织成JSON）、将目标函数体替换为`****EMPTY****`占位符的源代码文件、以及统一的执行模板。模型需补全缺失的实现。
    2.  **评估体系**：
        *   **双层分类法**：
            *   **通用3D能力 (General 3D Capability)**：
                *   几何变换 (Geometric Transformations, 24%)：坐标转换、投影等基础操作。
                *   力学/光学公式化 (Mechanics/Optics Formulation, 31%)：实现解析光学、力学方程等。
            *   **研究能力 (Research Capability)**：
                *   新颖算法实现 (Novel Algorithm Implementation, 34%)：实现论文提出的新思想。
                *   几何逻辑路由 (Geometric Logic Routing, 11%)：组合现有算子构建新流程。
        *   **单元测试生成**：为避免手动编写测试的开销，使用Cursor为每个目标函数自动生成10个覆盖默认参数、边缘情况和大规模输入的测试用例，并由专家审核以保证可靠性。
        *   **评分指标**：采用通过率 (`PassRate`)，即所有问题上单元测试通过率的平均值。

## 3. 实验设计
*   **基准测试 (Benchmark)**：GeoCodeBench本身即为本文提出的基准，包含100个高质量填空式代码补全问题。
*   **对比方法 (评估对象)**：评估了8个具有代表性的开源和闭源大语言模型：
    *   闭源模型: GPT-5, Claude-Sonnet-4.5, Gemini-2.5-Pro, Kimi-K2-Instruct
    *   开源模型: Doubao-Seed-1.6, Qwen3-Coder-480B, DeepSeek-R1, Llama-3.1-405B-Instruct
*   **实验场景/消融实验**：
    1.  **整体性能对比**：在所有100个问题上评估各模型的总体通过率及在四个子维度上的表现。
    2.  **上下文长度影响分析**：比较了三种输入条件对模型性能的影响：(a) 无论文文本；(b) 仅提供至“方法”部分的论文文本；(c) 提供完整论文文本。

##6 .主要结论与发现
1.	**能力存在巨大差距**:即使最佳模型GPT -5总体通过率也仅为36 .6%，表明当前LLMs距离可靠生成复杂三维科学代码仍有很长的路要走 。
2	.**创造性正确(Creative Correctness)**:部分成功案例中 ，模型采用了与原始实现不同但数学上等效的方法解决问题(例如分别使用基础矩阵F和本质矩阵E计算对极距离)，显示出真正的解决问题的能力 。
3	.**通用能力优于研究能力**:所有模型的“通用三维能力”得分均高于“研究能力”，两者呈正相关(Pearson r=0 .76)。这表明掌握几何基础是必要的 ，但不足以完成研究级的算法实现 。
4	.**上下文并非越多越好**:提供完整论文文本通常不会带来最佳性能 ，大多数模型在仅提供至“方法”部分的输入时表现最好 。这表明当前LLMs的长上下文科学理解能力有限 ，过多文本会引入噪声 。
5	.**主要失败模式**:功能性错误(代码可运行但逻辑错误)是最主要的失败类型 ，其次是张量形状和类型错误 。这反映了LLMs在多步几何推理和对工程语义(如边界条件 、参数约定)理解上的不足 。

##7 .优点
1.	.**高保真与现实性**:任务直接源于真实的前沿研究论文和代码库 ，准确反映了博士级研究的实际编码需求 。
2	.**自动化与可扩展性**:构建流程(论文解析 、函数提取 、测试生成)高度自动化 ，便于社区持续集成新的研究工作以更新基准 。
3	.**严谨的评估框架**:基于执行的单元测试确保了评估的客观性和可复现性；双层分类法有助于精细诊断模型的能力短板 。
4	.**深入的诊断分析**:不仅报告分数 ，还通过消融实验 、案例研究和错误分类等方式深入分析了失败原因 ，提供了宝贵的洞察 。

##8 .不足与局限
1	.**领域覆盖度**:虽然涵盖了高斯溅射 、NeRF 、SLAM等多个热门方向 ，但三维视觉领域极其广阔(如医疗影像 、机器人操控等)，基准的任务范围仍有扩展空间 。
2	.**任务形式单一**:目前仅限于“填空式”函数补全 。更复杂的任务形式(如从零开始实现整个模块 、修复bug 、多轮交互式开发)未被纳入 ，而这些也是实际研发中的常见场景 。
3	.**潜在的标注偏差**:尽管有自动化工具辅助 ，但核心函数的筛选和测试用例的最终审核依赖于人类专家的判断 ，可能引入主观偏差 。不过作者通过明确的筛选标准和专家审核流程尽力降低了此风险 。
4	.**静态评估环境**:评估在静态的沙箱环境中进行 。真实的研究编码往往涉及复杂的库依赖 、调试迭代和性能优化 ，这些动态方面未被考察 。
